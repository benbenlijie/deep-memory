# Internal notes

These documents are planning, positioning, launch, and metrics notes for
maintainers. They are not part of the user-facing documentation surface and
may reference targets, hypotheses, or competitive analysis that the public
README intentionally does not claim.

If you are a contributor looking for user-facing docs, start from the
[top-level README](../../README.md) or the docs under [`docs/`](../).

Contents:

- `POSITIONING.md` — product positioning, competitor truth table, proof obligations
- `README_REDESIGN.md` — research pass behind the current README information architecture
- `LAUNCH_PLAN.md`, `LAUNCH_LOOP.md`, `M1_GO_NO_GO.md`, `PHASED_KANBAN_PLAN.md` — launch and phased execution plans
- `METRICS.md`, `launch/weekly-metrics/` — growth metrics schema and weekly snapshots
- `WEBUI_SPEC.md`, `MEMORY_TO_SKILL.md`, `HERMES_INTEGRATION.md`, `extraction-contract.md` — feature specs and internal contracts
- `SMOKE_TRANSCRIPTS.md` — internal smoke test transcripts
- `TEAMBITION_BACKLOG.csv` — internal backlog snapshot

Claims made here must not surface in user-facing docs without going through
the proof obligations described in `POSITIONING.md`.
