02 — Module Map

模块结构

src/deep_memory/ 源码布局。所有模块围绕 core.py 构建。

核心引擎
core.py
DeepMemory 主类，MemoryRecord，SearchResult。SQLite 操作，搜索算法，生命周期管理，信任评分，备份恢复。
CLI 命令行
cli.py
入口：uv run deep-memory。命令：init, add, search, stats, export, webui, trust, scope, consolidate。
MCP Server
mcp_server.py
入口：uv run deep-memory-mcp。Stdio 工具：add, search, stats, resolve_conflict, conflicts, memory_feedback。
WebUI
webui.py
记录浏览器，关系图可视化（vis.js），时间线视图，洞察分析面板（检索质量、反馈统计）。
会话提取
extraction.py
AgentMessage，ConversationExtractionInput，ExtractedMemory。基于规则的确定性提取（v0），将对话转为结构化记忆。
便携同步
portable.py
Export: memories.jsonl + manifest.json。Import: 合并语义，去重，冲突解决。diff_databases() 跨设备比较。
技能导出
skill_export.py
SkillCandidate 类。程序性记忆 → 可审核 skill markdown。含触发条件、边界检查、验证步骤、人类审核清单。
隐私策略
privacy.py
MemoryPolicyDecision (ALLOW/DENY/REQUIRES_CONFIRMATION)。ensure_memory_content_allowed() 统一安全守卫。
Hermes 适配器
adapters/hermes.py
Hermes 会话事实 JSONL 导入。支持 user/tenant/workspace 作用域，信任等级追踪。
代理包装器
adapters/agent_wrapper.py
Codex/OpenCode 包装器。任务前注入 DEEP_MEMORY_CONTEXT，任务后导入 facts-out JSONL。
依赖关系：所有模块以 core.py 为枢纽。cli.py 和 mcp_server.py 是主入口点。adapters/、portable.py、extraction.py、skill_export.py 全部调用 core.py 的 DeepMemory 类。privacy.py 是无外部依赖的独立安全层，被所有写入路径调用。
03 — Memory Lifecycle

记忆生命周期

记忆是有生命周期的状态，不是只追加的日志。

candidate
候选 · 待审核
→
active
活跃 · 默认召回
→
superseded
被替代
deprecated
软删除
archived
合并归档

类型衰变半衰期 · Decay Half-lives

working 工作记忆
7 天
episodic 情节记忆
30 天
semantic 语义记忆
180 天
procedural 程序记忆
365 天
合并机制 · Consolidation：当同类 active 记录的 token 重叠相似度 > 0.6 时，系统创建一条摘要记录（取组内最大 importance，平均 confidence，标注来源 "consolidated from <ids>"），将原始记录标记为 archived 并指向摘要。破坏性操作前自动创建 7 天 TTL 的本地备份。
04 — Trust Model

信任模型

source 不只是字符串注释——而是可计算、可审计、可降权的安全边界。

1.0
user
用户直接表达、审核确认后的偏好或事实 · 自动触发：agent=human/user, origin=explicit
0.9
verified
经测试、人审、权威来源核验 · 手动提升：trust promote --to verified
0.8
agent-high
高质量自动写入，有策略约束和可审计来源 · 可信代理显式传入
0.5
agent-auto
普通 agent 自动提取 · 默认 fallback，旧 plain string source 兼容
0.5
external
网页抓取、外部导入 · URL/地址自动检测 (http/ftp/mailto/IP)
0.2
untrusted
明确可疑、恶意、隔离区内容 · 用于保留证据但强降权
检索权重公式 · Retrieval Weighting
base_score = lexical * 0.55 + importance * 0.25 + confidence * 0.15 + decay * 0.05
final_score = base_score * trust_factor
冲突防御：当新写入的低信任记忆与已有的高信任 active 记忆冲突（token overlap > 0.5，kind 不同），新记忆不会直接进入 active 状态，而是标记为 candidate，等待人工审核。这防止了记忆投毒——低信任导入无法通过高 importance/confidence 覆盖用户确认的事实。
05 — Write Policy

写入策略

deep-memory 不设计成记住一切。每条候选记忆被分为三类。

✓ allow
安全、持久、有用的事实。
允许写入。
✗ deny
过于风险、原始或误导。
拒绝写入。
! confirm
可能合法但需明确批准。
仅允许同意感知路径。

✓ 默认允许

  • ✓ 用户偏好：语言、语调、工具选择
  • ✓ 项目约定：构建命令、编码风格、架构决策
  • ✓ 已验证的程序性事实：测试通过的工作流
  • ✓ 稳定的非敏感领域事实
  • ✓ 不涉及私人档案的团队事实

✗ 默认拒绝

  • ✗ 密码、API 密钥、OAuth 令牌、私钥
  • ✗ 完整聊天记录、邮件、DM、工具日志
  • ✗ PR 号、issue 号、commit SHA、任务 ID
  • ✗ 凭证重放、钓鱼、恶意软件
  • ✗ 对身份、健康、财务、关系的未验证推测

跨代理适配

四种 AI 代理通过统一协议连接到同一个本地数据库。

写：MCP add 在测试/审核/用户确认后

Codex
Wrapper MVP 已实现 ✅
读：包装器预置召回块 (DEEP_MEMORY_CONTEXT)
写：运行后 JSONL facts 导入，需证据
deep-memory codex-run --db ... --task ... --facts-out ...
OpenCode / OpenClaw
规范 · Spec
读：运行前召回或循环内 MCP
写：检查点触发的显式事实导入
future: deep-memory opencode-run --db ...
最小共同协议：每个适配器只做两件事——任务前/中 search，任务后 add 已验证的持久事实。无隐藏转录抓取，无静默云同步，无自动安装技能。

08 — Dual-Temporal & Provenance

双时态与溯源

每条记忆有两条独立的时间线，支持时间旅行查询。

event_time — 事实为真 / 事件发生的时间
例：项目约定在 3 月 1 日确立
learned_at — deep-memory 写入记录的时间
例：你在 3 月 15 日才把这个约定写入记忆
"上周我对这个项目了解什么？"
使用 --as-of <date> 查询特定事件时间点的有效记忆。筛选条件：event_time ≤ as_of AND (valid_until IS NULL OR valid_until > as_of)。
"偏好何时改变的？"
通过冲突链追踪变更历史。valid_until 在冲突解决时自动设置，标记旧记忆失效时间。
"导入时间是否晚于事件发生时间？"
数据质量审计。如果 learned_at 远晚于 event_time，说明是外部导入的历史数据，需检查可信度。
冲突链 · Conflict Chain
解决冲突时，系统自动维护血统关系，保留完整审计轨迹。

血统字段 · Provenance Fields

supersedes_id → "这条记录替代了那条旧记录"
superseded_by_id → "那条更新的记录替代了我"
valid_until → 冲突解决时自动设置，标记事实失效时间点
